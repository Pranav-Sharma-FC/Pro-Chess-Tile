## Sprint 1
For sprint 1, we worked on nailing down the totally fundamental fundamentals of the game, like having an actual chess board, having actual pieces that can move, and lining up coordinates with the logic of the game; plus other basic stuff.  
### Work Done
I worked on a small portion of piece logic and then started working on how we're going to have our HUD and/or UI going, and what we're going to do with all of our extra abilities, and the currencies we have in our game. And just because I felt like it, when I'm not doing UI/HUD things, I'm playing around with the sound effects of the game. 
### Difficulties
The logic of our game is a little wonky. We have an idea and a good "concept" of everything we want to do but since there are so many parts to our game, we have to make sure we have everything actually playable instead of a bunch of abstract concepts sort of put together in a game that we can't really use. 
### Surprises
How interconnected everything in our game is. When we were dividing up the work, we realized how the chessboard and the logic with the pieces basically go completely hand in hand. And creating the actual pieces is just a sprite and a collision shape. 
### Lessons
I definetly think we should have thought more about the actual coding part of our game. The concept of this game is really cool and it's challenging but not impossible, but it's definetly a little bit more difficult than necessary as when we were planning we talked about elemenets of the game and ideas instead of how we're actually going to code it.
